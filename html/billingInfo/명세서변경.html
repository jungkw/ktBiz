이용안내

e-Mail명세서는 매월 고객님께 사용하신 이용요금을 통하여 받아 보실 수 있도록 암호화 하여 보내집니다. 보안 메일을 원하지 않으시면 암호화 해지된 메일을 보내드립니다.

암호화된 명세서의 요금 내역을 보시려면 먼저 전용 Plug-In을 설치하셔야 하며(최초 1회만 설치, 자동으로 설치됨), 이미 설치되어 있는 경우는 별도 설치가 필요 없습니다.

e-Mail명세서는 소중한 고객님의 정보를 보호하기 위하여 개인이신 경우 고객님의 주민등록번호 뒷자리 7자리, 사업자이신 경우 사업자등록번호 뒷자리 5자리를 입력하여 보실 수 있도록 암호화되어 있습니다.

e-Mail명세서를 인쇄하실 때에는 명세서 전용 인쇄 Plug-In을 설치하신 후 명세서에서 책갈피에 있는 인쇄버튼을 클리하시어 인쇄하시기 바랍니다.

명세서 전용 Plug-In을 설치하셔야 하는 경우

  • - 컴퓨터를 포맷하신 후 암호화 된 메일을 보실 경우
  • - e-Mail 명세서 전용 Plug-In을 삭제하신 경우
  • - e-Mail 명세서를 처음으로 받으시는 경우 등

보안메일 보는 방법

Active X 설치를 위한 설정

Mail 보기 버튼 위치에 빨간색 X표시만 보일 경우에는 명세서 암호화 Plug-In 수동설치를 해주시기 바랍니다.

윈도우 좌하단 [시작]-[설정]-[제어판]-[인터넷옵션]-[보안탭]에서 이 영역에 적용할 보안 수준을 기본수준으로 변경하신 후 확인해 주시기 바랍니다.

인터넷 익스플로러 실행 후 [도구]-[인터넷 옵션]-[고급]-[멀티미디어]에서 그림표시를 체크 하시고, [고급]-[탐색]에서 URL 항상 UTF-8로 보냄에 체크가 되어 있으면 체크를 풀어주신 후 반드시 윈도우 재부팅하신 후 확인하시기 바랍니다.

e-Mail명세서를 여신 후 상단의 상태표시줄을 클릭 하신 후 Active X 컨트롤 설치를 선택하시기 바랍니다.

보안 경고창이 뜨면 설치를 선택 하시기 바랍니다.

웹아웃룩 설정

웹아웃룩에서 e-Mail명세서 보안메일을 볼수 없을 경우에는 웹페이지로 보기를 클릭하시면 첨부파일 형태인 e-Mail명세서 보안메일을 여실 수 있습니다.

아웃룩익스프레스 설정

아웃룩 익스프레스에서 명세서를 열 때 Internet Explorer창의 주소 아래에 아래와 같은 문구가 보이며 명세서가 열리지 않을 경우, "보안 설정이 웹 사이트에서 컴퓨터에 설치된 ActiveX 컨트롤을 실행하는것을 허용하지 않기 때문에..."

아웃룩 익스프레스를 열고 [도구]-[옵션]-]보안]에서 인터넷(보안수준은 낮고, 기능성 높음)(Z)에 체크하시고 바이러스 가능성이 있는 첨부 파일을 저장하거나 열수 없음(N)에 체크를 제거하신후 명세서를 확인하시기 바랍니다.

발송여부 선택이란?

명세서 발송

이메일 명세서 변경 신청 시 입력하신 이메일 주소로 이메일 명세서 신청 변경 완료 메일을 보내 드립니다.

명세서 발송제외

이메일 명세서 변경을 신청하셨지만, 이메일이 아닌 웹을 통해 청구 내역 확인하기를 희망하실 경우 선택하시면 이메일 명세서 변경을 발송하지 않습니다.

  • 명세서 발송제외를 선택 하시더라도 이메일 명세서 변경 신청에 따른 혜택은 모두 받으실 수 있습니다.
  • 지로로 요금을 납부하시는 고객님은 이메일 명세서 변경을 신청하시더라도 명세서 발송제외 선택을 하실 수 없습니다.
  • 명세서 발송제외 시 스마트명세서(MMS)는 이용하실 수 없습니다.
  • 이메일 명세서로 신청하셨지만 이메일이 아닌 사이트를 통해서 명세서 확인을 원할 경우 명세서 발송제외를 선택하시면 이메일명세서는 발송되지 않습니다.
  • My페이지 > 요금 조회/납부 > 요금납부/조회 > 요금납부 에서 납부하실 내역을 확인하실 수 있습니다.
  • 이메일 명세서를 신청하시게 되면 신청 다음달부터 발송되며, 매일 17일 전후로 이메일을 통해 명세서를 보내드리고 있습니다. (신청시점이 언제이신지 꼭 확인해 주세요.)
  • 지로로 요금을 납부하시는 고객님은 이메일 명세서 변경을 신청하시더라도 명세서 발송 제외 선택을 하실 수 없습니다.

발송오류안내

네트워크 에러
네트워크 에러는 송,수신쪽의 네트워크 장애로 발송 실패되는 경우입니다.
메일 저장공간 부족
메일을 수신/저장하는 시스템의 스토리지에 공간이 부족한 경우에 발생합니다. 이에 대한 DSN코드 값은 452번과 552번 코드를 반환하게 됩니다. 452번의 경우 메일 서버가 동작하는 시스템의 저장공간이 부족한 경우에 반환되며, 552번의 경우 사용자에게 할당된 저장공간이 초과된 경우에 발생합니다.
존재하지 않는 도메인
이메일 주소에 지정된 도메인이 실제로 존재하지 않는 경우에 해당됩니다. 예를 들어, aaa.co.kk 와 같이 도메인 구조에 맞지 않는 주소가 입력된 경우나 m00-00m.co.kr 처럼 도메인 자체가 아예 등록되어있지 않은 경우에 발생합니다.
계정 없음
실제 도메인은 존재하나 해당 메일 수신업체에 사용자 계정이 없는 경우입니다.
휴먼계정
도메인과 사용자 계정이 존재하나 수신업체에서 해당 계정을 휴면계정으로 처리하는 경우에 해당합니다.
정의되지 않는 에러
메일 수신업체로부터 받은 발송에러 메세지 내용이 모호하여 명확하지 않는 경우입니다.
메일주소 오류
회원 가입시에 기록하게 되어있는 이메일 주소의 규칙이 맞지 않는 경우에 발생하는 에러입니다. 예를 들어, 아이디와 도메인을 구분하는 @표기가 빠져있다던가, 또는 @표기가 여러 번 중복되어 표시되는 경우, 도메인 주소에 '.' 대신 ',' 가 들어있는 경우 등 입니다.
도메인 네임서버 찾지 못함
메일 발송서버가 네트워크를 통해 수신 메일서버를 찾아가는 과정에서 수신 메일서버의 정확한 위치를 알려주는 도메인 네임 서버를 먼저 찾게 되는데 이 도메인 네임서버를 찾지 못하는 경우 입니다. 이 에러는 해당 도메인 네임 서버가 일시 기능정지 되었을 때도 발생할 수 있습니다.
메일전송 중에 전송이 중단된 경우
메일 발송 서버와 메일 수신 서버간의 메일을 전달하는 단계에서 서버가 연결이 끊긴다거나 트랙잭션이 발생하여 더이상 메일 전달이 안되는 경우입니다.
수신 메일서버 없음
도메인 정보를 알려주는 도메인 네임 서버에 해당 수신 도메인은 존재하나 수신 메일서버 정보가 없을 경우로 인해 발송서버가 수신 메일서버를 찾지 못하는 경우입니다.
스팸처리, 수신거부
수신 서버에서 발송서버를 스팸서버로 인지하여 수신을 하지 않고 BLOCKING 하는 경우와 고객께서 수신 이메일 설정에서 발송서버 정보를 수신거부나 스팸 설정으로 등록하여 결과적으로 수신이 되지 않는 경우입니다.
메일 서비스 중지 도메인
메일서비스를 하던 도메인에서 메일 서비스를 중지한다는 정보를 입수 했을 경우, 즉 100% 메일 발송에 실패나는 경우라고 판단될 경우를 말합니다. 메일서비스를 중지한 메일주소를 가지고 고객님께서는 정상 발송 될 수 있도록 다른 메일주소로 고객센터에 문의하셔서 수정하셔야 합니다.
